# MoreTechHackathon

MoreTechHackathon - это проект, выполненный в рамках хакатона MoreTech 5.0 от ВТБ. Проект представляет собой нативное Android-приложение, написанное на Kotlin, ряд актуальных разработок и идей, а также [макет в Figma](https://www.figma.com/file/1w5hXkOTOeNablSHlg5q81/MoreTech-5.0-Mobile-App?type=design&node-id=0%3A1&mode=design&t=bR4LwE8XScfJtRan-1).
Backend к проекту: https://github.com/StaZisS/VTB-Tech/tree/master

Запустить проект:
```git clone https://github.com/Lvntt/MoreTechHackathon.git```

# Задача

Требования:
+ Функциональные
    + Отображение карты города, в котором находится пользователь, включая метку его текущего местоположения и метки отделений Банка ВТБ
    + Возможность построения как пешего, так и автомобильного (опционально на общественном транспорте) маршрута до подходящего отделения
    + Поиск оптимального отделения с учетом его загруженности, а также задачи клиента на основе собственного алгоритма
    + Дополнительные "вау-фичи"
+ Нефункциональные
    + Авторский алгоритм определения загруженности отделения банка
    + Удобный интерфейс выбора отделения под заданные критерии
    + Безопасность
    + Масштабируемость
    + Стилистика Банка ВТБ
 
В рамках разработки проекта выполнены все функциональные и нефункциональные требования. Добавленные "вау-фичи": чат-бот, история поиска.

### Основной функционал
В приложении в качестве основного функционала реализована возможность просмотра карты с кластерами банкоматов/отделений банка, а также возможность просмотреть информацию о конкретном банкомате или отделении, включая его загруженность, часы работы для юридических и физических лиц, возможности для людей с ограничениями жизнедеятельности, а также ближайшие станции метро.
На экране с подробной информацией можно нажать кнопку "Проложить маршрут" и посмотреть автомобильный и пеший маршрут от текущего местоположения пользователя до выбранного отделения банка или банкомата.
Пользователь также может посмотреть рейтинг наиболее подходящих для него отделений банка или банкоматов с учетом выбранных услуг (если они были выбраны), либо без их учета. При составлении рейтинга учитывается расстояние от клиента до отделения, а также загруженность банка по выбранным услугам.

### Инклюзивность
В идее приложения предусмотрена возможность более удобного взаимодействия с сервисом для людей с ограчничениями жизнедеятельности. После этапа выбора услуг, которые интересуют пользователя, ему предлагается выбрать что-то из опций "Нарушения зрения", "Ограничения в передвижении", либо отсутствие ограничений. Выбор пользователя впоследствии будет учитываться в подборе подходящих отделений/банкоматов, и пользователь будет получать только те, которые предоставляют возможность пользования с ограниченными возможностями.

### Чат-бот
Идея создания чат-бота состоит в том, чтобы пользователь мог использовать функционал приложения, применяя запросы на естественном языке. Чтобы добиться приближенного к нужному результата, на Backend части приложения сделан сервис чат-бота, обрабатывающий запросы с использованием алгоритма расстояния Левенштейна.

### История поиска
История поиска помогает пользователю вспомнить, какие отделения или банкоматы он искал ранее. Отделение или банкомат фиксируются в истории поиска, если что-то из этого было выбрано, а затем к ним был построен маршрут.
